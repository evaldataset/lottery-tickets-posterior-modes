# New Asset Inventory

Date: 2026-05-07

This inventory lists project-created assets introduced by the paper and local
artifact package. It complements `docs/public_release_manifest.md`, whose JSON
sidecar records per-file SHA256 hashes for the package contents.

## Asset Classes

| Asset class | Representative paths | Generated by | Included in local release manifest | Notes |
| --- | --- | --- | --- | --- |

## Public Release Status

The local asset package is documented and hash-inventoried. It has not yet
been uploaded to an anonymized public repository or supplementary-material
system, but a local tarball and SHA256 sidecar are generated by
`make release-archive`. The project license is recorded in the top-level
`LICENSE` file, using "Anonymous Authors" for the anonymous-review package.
External upload and public repository state are tracked separately by
`docs/external_validation_receipts.json`,
`docs/public_repository_snapshot_audit.md`,
`docs/external_validation_readiness_audit.md`,
`docs/external_validation_runbook.md`,
`docs/venue_submission_compliance_audit.md`,
`docs/submission_readiness_audit.md`, and
`docs/thread_goal_completion_audit.md`.

## Verification

The asset inventory is considered current only when these commands pass:

```bash
make check
make paper-check
make paper-neurips-check
make release-archive
make release-archive-smoke
make public-repository-snapshot
make external-validation-readiness
make container-check
```

`scripts/verify_research_artifacts.py` checks that this file, the compute
resource accounting, the asset license inventory, and the generated public
release manifest/archive are present and internally consistent with the
expected release boundary.
